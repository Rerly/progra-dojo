    上級① クロージャとは？

    JavaScript 
    
function counter() {
  let count = 0;

  return function() {
    count++;
    return count;
  };
}

const myCounter = counter();

console.log(myCounter());
console.log(myCounter());
console.log(myCounter());

    考えてみよう

    なぜ count は毎回リセットされないのか？

    解説

    counter() の中の count は本来なら関数終了と同時に消える。 しかし、内部の関数が count を参照しているため、 JavaScriptはその変数を保持し続ける。

    この「外側の変数を覚えている関数」の仕組みを クロージャ（closure）という。

    上級② モジュールパターン

    JavaScript 
    
const CounterModule = (function() {

  let count = 0;

  function increment() {
    count++;
    return count;
  }

  function reset() {
    count = 0;
  }

  return {
    increment,
    reset
  };

})();

console.log(CounterModule.increment());
console.log(CounterModule.increment());
CounterModule.reset();
console.log(CounterModule.increment());

    何が起きている？

    count は外から直接アクセスできない。 しかし increment と reset を通して操作できる。

    ポイント

    即時関数（IIFE）でスコープを作り、 外に公開するものだけ return している。 これが「モジュールパターン」。

    上級③ this の正体

    JavaScript

    解説

    this は「その関数を呼び出したオブジェクト」を指す。 今回は user が greet を呼び出しているので、 this は user を指す。

    上級⑨ map()

    JavaScript 
    
const numbers = [1, 2, 3];

const doubled = numbers.map(num => num * 2);

console.log(doubled);

    このコードは何をしている？

    numbers 配列の各要素を2倍に変換し、 新しい配列を作っている。

    処理の流れ

    • 1 → 2
    • 2 → 4
    • 3 → 6

    map() は配列の長さを変えずに、 全要素を変換するメソッド。

    ポイント

    • 元の配列は変更されない
    • 必ず新しい配列を返す
    • 配列の長さは同じ

    上級⑩ filter()

    JavaScript 
    
const numbers = [1, 2, 3, 4, 5];

const even = numbers.filter(num => num % 2 === 0);

console.log(even);

    このコードは何をしている？

    numbers 配列から「偶数だけ」を取り出して、 新しい配列を作っている。

    処理の流れ

    • 1 → 奇数 → 除外
    • 2 → 偶数 → 残す
    • 3 → 奇数 → 除外
    • 4 → 偶数 → 残す
    • 5 → 奇数 → 除外

    条件が true のものだけが、新しい配列に入る。

    ポイント

    • 条件に合う要素だけ残す
    • 元の配列は変更されない
    • 配列の長さは減ることがある

    上級⑪ reduce()

    JavaScript 
    
const numbers = [1, 2, 3, 4];

const sum = numbers.reduce((acc, curr) => {
  return acc + curr;
}, 0);

console.log(sum);

    このコードは何をしている？

    numbers 配列のすべての値を合計している。

    処理の流れ

    初期値 acc = 0

    • 1回目 → 0 + 1 = 1
    • 2回目 → 1 + 2 = 3
    • 3回目 → 3 + 3 = 6
    • 4回目 → 6 + 4 = 10

    最終的に 10 が返る。

    ポイント

    • acc は累積値（今までの結果）
    • curr は現在の値
    • 初期値を指定できる
    • 配列を1つの値にまとめる

    上級⑫ map / filter / reduce 総まとめ

    3つの役割の違い

    メソッド 何をする？ 結果
    map 各要素を変換する 同じ長さの新しい配列
    filter 条件に合うものだけ残す 短くなることがある配列
    reduce 配列を1つにまとめる 1つの値

    イメージで理解

    map → 変換工場

    filter → ふるい

    reduce → まとめ役

async function loadPosts() {
  try {
    const res = await fetch("https://jsonplaceholder.typicode.com/posts");

    if (!res.ok) {
      throw new Error("通信エラー");
    }

    const posts = await res.json();

    const result = posts
      .filter(post => post.userId <= 3)
      .map(post => ({

        acc[post.userId] += post.titleLength;
        return acc;
      }, {});

  } catch (error) {
    console.error("エラー発生:", error);
  }

    最終② 並列API集計システム

    JavaScript 
    
async function loadDashboard() {
  try {
    const [usersRes, postsRes, commentsRes] = await Promise.all([
      fetch("https://jsonplaceholder.typicode.com/users"),
      fetch("https://jsonplaceholder.typicode.com/posts"),
      fetch("https://jsonplaceholder.typicode.com/comments")
    ]);

    if (!usersRes.ok || !postsRes.ok || !commentsRes.ok) {
      throw new Error("通信失敗");
    }

    const users = await usersRes.json();
    const posts = await postsRes.json();
    const comments = await commentsRes.json();

    const dashboard = users.reduce((acc, user) => {

      const userPosts = posts.filter(post => post.userId === user.id);

      const totalComments = userPosts.reduce((sum, post) => {
        const commentCount = comments.filter(
          comment => comment.postId === post.id
        ).length;

        return sum + commentCount;
      }, 0);

      acc[user.name] = {
        postCount: userPosts.length,
        commentCount: totalComments
      };

      return acc;

    }, {});

    console.log(dashboard);

  } catch (error) {
    console.error("重大エラー:", error);
  }
}

loadDashboard();

    処理の全体像

    ① Promise.all
    users / posts / comments を並列取得している。

    ② users を reduce
    各ユーザーごとにダッシュボード情報を作成。

    ③ 投稿抽出
    post.userId === user.id で、その人の投稿だけ取得。

    ④ コメント集計
    各投稿に紐づくコメント数を合計。

    ⑤ 動的キー生成
    acc[user.name] でオブジェクトを構築。

    最終的な形：

    {
  "ユーザー名": {
    postCount: 投稿数,
    commentCount: コメント合計
  }
}

    最終③ 高速最適化版

    JavaScript 
    
async function loadDashboardFast() {
  try {
    const [usersRes, postsRes, commentsRes] = await Promise.all([
      fetch("https://jsonplaceholder.typicode.com/users"),
      fetch("https://jsonplaceholder.typicode.com/posts"),
      fetch("https://jsonplaceholder.typicode.com/comments")
    ]);

    const users = await usersRes.json();
    const posts = await postsRes.json();
    const comments = await commentsRes.json();

    // ① postIdごとのコメント数を事前集計
    const commentCountMap = comments.reduce((acc, comment) => {
      acc[comment.postId] = (acc[comment.postId] || 0) + 1;
      return acc;
    }, {});

    // ② userIdごとの投稿を事前集計
    const postMap = posts.reduce((acc, post) => {
      if (!acc[post.userId]) acc[post.userId] = [];
      acc[post.userId].push(post);
      return acc;
    }, {});

    // ③ 最終ダッシュボード生成
    const dashboard = users.reduce((acc, user) => {

      const userPosts = postMap[user.id] || [];

      const totalComments = userPosts.reduce((sum, post) => {
        return sum + (commentCountMap[post.id] || 0);
      }, 0);

      acc[user.name] = {
        postCount: userPosts.length,
        commentCount: totalComments
      };

      return acc;

    }, {});

    console.log(dashboard);

  } catch (error) {
    console.error(error);
  }
}

loadDashboardFast();

    なぜこのコードは速いのか？

    ① 事前集計（インデックス作成）
    comments を reduce して、 postIdごとのコメント数を先にまとめている。

    ② 投稿も事前グループ化
    posts を reduce して、 userIdごとの配列にまとめている。

    ③ ループの中で filter を使わない
    毎回全体検索せず、 オブジェクトから直接参照している。

    ④ 計算量の違い
    前のコードは O(n²) 以上。
    このコードは O(n)。

    つまり：
    データが1万件でもスケールする設計になっている。
